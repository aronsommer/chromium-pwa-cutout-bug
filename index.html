PWA Camera Cutout Bug – Chromium

On Android devices with a camera cutout (punch-hole), a PWA with display: fullscreen in the manifest shows a black bar in the cutout area even when viewport-fit=cover is set. The same app using requestFullscreen() via JavaScript renders correctly behind the cutout.

Install both PWAs and compare

✅ Working – display: standalone + JS requestFullscreen() After first tap, content renders correctly behind the camera cutout. ❌ Broken – display: fullscreen (manifest only) Black bar covers the camera cutout area despite viewport-fit=cover.

Root cause (suspected)

requestFullscreen() sets LAYOUT_IN_DISPLAY_CUTOUT_MODE_SHORT_EDGES on the Android window, allowing content to render behind the cutout. Manifest display: fullscreen uses a different cutout mode that blocks rendering there, making viewport-fit=cover ineffective.
